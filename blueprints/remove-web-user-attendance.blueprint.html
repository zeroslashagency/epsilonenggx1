# Remove "Web User Attendance", keep "Mobile User Attendance"

Source: live request on /settings/roles/<id>/edit

## Outcome
The role permission editor shows two near-duplicate attendance groups. Remove the "Web User Attendance" group so only "Mobile User Attendance" remains.

Key risk: 'web_user_attendance' is a live permission key read at runtime in 15 places in app-sidebar.tsx and 9 in fir/page.tsx. Deleting only the data block breaks web FIR/Calls/Device-Monitoring nav + the FIR tool page. Block removal MUST be paired with repointing those gates to 'mobile_user_attendance' (which already contains the same FIR + Device Monitoring item names).

- app/(app)/settings/roles/[id]/edit/page.tsx — drop web_user_attendance legacy hydration (~168-169)
- components/app-sidebar.tsx — repoint 15 hasPermission('web_user_attendance', ...) -> 'mobile_user_attendance'
- app/(tools)/fir/page.tsx — repoint 9 refs
- app/lib/features/auth/permission-mapping.ts, permission-checker.ts, auth-context.tsx — remove web_user_attendance branches, keep mobile + user_attendance legacy alias

## Open questions
Q1: scope — recommended: delete block + repoint gates.
Q2: migrate existing role JSON web_user_attendance.* -> mobile — recommended: merge on save.
Q3: Calls items (web-only) — recommended: drop with web group.

Outcome & delta

↻ Reuses (no new infra)

  • ›mobile_user_attendance — kept as the single attendance group; already contains FIR + Device Monitoring items
  • ›permission-mapping.ts maps — MOBILE_USER_ATTENDANCE_MAP stays the source of codes
  • ›The editor table renderer (Object.entries(permissionModules).map) — unchanged; it just renders one fewer group

− Removed / repointed

  • −web_user_attendance block deleted from permissionData.ts
  • ±Sidebar + FIR page gates repointed from 'web_user_attendance' → 'mobile_user_attendance'
  • ±Legacy user_attendance normalization now hydrates only the mobile group

Why not just delete the block? The string 'web_user_attendance' is a live permission key read at runtime in 15 places in app-sidebar.tsx and 9 in fir/page.tsx. Deleting only the data block would leave those hasPermission('web_user_attendance', …) calls always-false → the web FIR / Calls / Device-Monitoring nav items and the FIR tool page silently disappear for every non-super-admin. So the block removal must be paired with repointing those gates.
